MRRealmResultsControllerDelegate Protocol Reference

Conforms to NSObject
Declared in MRRealmResultsController.h

Overview

An instance of MRRealmResultsController uses methods in this protocol to notify its delegate that the controller’s sections have been changed.

Tasks

  • – controller:didChangeSection:atIndex:forChangeType:

    Notifies the delegate of added and removed sections and sections that have changed their number of objects.
  • – controllerWillChangeContent:

    Notifies the delegate that section changes are about to be processed and notifications will be sent.
  • – controllerDidChangeContent:

    Notifies the delegate that all section changes have been sent.
  • – controller:sectionIndexTitleForSectionName:

    Asks the delegate to return the corresponding section index title for a given section name.

controller:didChangeSection:atIndex:forChangeType:

Notifies the delegate of added and removed sections and sections that have changed their number of objects.

- (void)controller:(MRRealmResultsController *)controller didChangeSection:(id<MRRealmSectionInfo>)sectionInfo atIndex:(NSUInteger)sectionIndex forChangeType:(MRRealmResultsChangeType)type

Parameters

Controller instance that noticed the change on its sections.

sectionInfo

type

Indicates if the change was an insert, delete or count change.

index

Index of changed section.

Discussion
